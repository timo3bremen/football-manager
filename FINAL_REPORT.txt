AUTHENTIFIZIERUNG
=================

GET /api/auth/leagues
  → Alle 7 Ligen mit Metadaten
  Response: Array<LeagueInfoDTO>
  Beispiel: [
    { id: 1, name: "1. Liga", division: 1, divisionLabel: "1. Liga", 
      filledSlots: 12, totalSlots: 12 },
    { id: 2, name: "2. Liga A", division: 2, divisionLabel: "2. Liga A",
      filledSlots: 11, totalSlots: 12 },
    ...
  ]

POST /api/auth/register-with-league
  Body: { username, password, teamName, leagueId }
  → Registrierung mit Liga-Wahl
  Response: { token, teamId }

SPIELPLAN
=========

GET /api/v2/schedule/leagues
  → Alle Ligen für Dropdown
  Response: Array<LeagueInfoDTO>

GET /api/v2/schedule/standings
  → User-Liga Tabelle (Standard)
  Response: Array<LeagueStandingsDTO>

GET /api/v2/schedule/standings/league/{leagueId}
  → Liga-spezifische Tabelle
  Response: Array<LeagueStandingsDTO>
  Beispiel Response: [
    { teamId: 5, teamName: "Bayern", position: 1, played: 10, won: 8,
      drawn: 1, lost: 1, goalsFor: 28, goalsAgainst: 8, points: 25, strength: 445 },
    ...
  ]

ADMIN
=====

POST /api/admin/initialize-leagues
  → Manuell Ligen initialisieren (meist nicht nötig)
  Response: { success: true, message: "..." }

║                        TROUBLESHOOTING                                       ║
╚══════════════════════════════════════════════════════════════════════════════╝

Problem: Ligen werden nicht initialisiert
Lösung:  POST /api/admin/initialize-leagues aufrufen
         oder DELETE /api/admin/clear-all dann Server neustarten

Problem: Tabelle zeigt keine Teams
Lösung:  Überprüfe ob selectedLeague != null
         Check: GET /api/v2/schedule/standings/league/1

Problem: Frontend wechselt Liga nicht
Lösung:  Browser-Console überprüfen auf Fehler
         Check Network-Tab: GET /api/v2/schedule/standings/league/{id}

Problem: leagueId nicht in localStorage
Lösung:  Re-register oder POST /api/auth/register-with-league debuggen
