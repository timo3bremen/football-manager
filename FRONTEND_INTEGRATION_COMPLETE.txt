✅ API-INTEGRATION
==================

Neue API-Calls:

Auth.jsx:
  GET  /api/auth/leagues
       → Alle verfügbaren Ligen abrufen
       Response: [{ id, name, division, divisionLabel, filledSlots, totalSlots }, ...]

  POST /api/auth/register-with-league
       Body: { username, password, teamName, leagueId }
       Response: { token, teamId }

Schedule.jsx:
  GET  /api/v2/schedule/leagues
       → Alle Ligen (für Dropdown)
       Response: [{ id, name, division, divisionLabel, filledSlots, totalSlots }, ...]

  GET  /api/v2/schedule/standings
       → User-Liga Tabelle (Default)
       Response: [{ teamId, teamName, position, played, won, drawn, lost, goalsFor, goalsAgainst, points, strength }, ...]

  GET  /api/v2/schedule/standings/league/{leagueId}
       → Spezifische Liga Tabelle
       Response: Wie oben

✅ USER-FLOW
=============

Registrierung:
  1. User öffnet Registrierungsformular
  2. Backend lädt verfügbare Ligen
  3. User wählt Liga aus Dropdown
  4. User füllt Benutzername, Passwort, Teamname
  5. User klickt "Registrieren"
  6. Backend:
     - Erstellt neues Team
     - Findet Liga
     - Findet zufälligen CPU-Slot
     - Ersetzt CPU-Team mit User-Team
     - Erstellt User mit leagueId
  7. User wird eingeloggt
  8. leagueId wird in localStorage gespeichert

Spielplan-Ansicht:
  1. User öffnet Spielplan
  2. Header zeigt: "📌 Deine Liga: [Liga Name]"
  3. Im Tabelle-Tab:
     - Dropdown zeigt aktuelle Liga (mit allen verfügbaren)
     - Tabelle zeigt nur Teams aus dieser Liga
  4. User klickt auf Liga im Dropdown
  5. Tabelle wird neu geladen
  6. selectedLeague & userLeagueLabel werden aktualisiert
  7. Spielplan-Tab zeigt immer User-Liga
