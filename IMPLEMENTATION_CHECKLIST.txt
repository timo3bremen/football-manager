✅ WORKFLOW
-----------
1. Server startet
   → ApplicationStartupListener.onApplicationReady()
   → RepositoryService.initializeLigues()
   → 7 Ligen + 84 CPU-Teams erstellt

2. User registriert sich
   → GET /api/auth/leagues (zeigt Liga-Optionen)
   → POST /api/auth/register-with-league
   → Zufälliger CPU-Slot wird mit User-Team ersetzt

3. User sieht Spielplan
   → GET /api/v2/schedule/standings (eigene Liga)
   → GET /api/v2/schedule/leagues (Liga-Auswahl)
   → GET /api/v2/schedule/standings/league/{leagueId} (andere Liga)
